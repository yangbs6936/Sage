## 章节标题

从单点场景到企业级 Agent 平台

### 讲解思路

现有 PPT 已经讲清楚了 CMB 在客户沟通、RM Copilot、AI Xiaozhao、CRM 嵌入、IM 外呼线索筛选等场景上的实践。新增章节建议把叙事往平台层抽象：这些不是单个 demo，而是由 Sage 支撑的一套可复用、可扩展、可治理的 Agent 平台能力。

## 第 1 页

### 标题

Sage 将 Agent 试点转化为可复制的银行产品能力

### 核心信息

很多 Agent 试点停留在单一场景。Sage 的价值在于，把成功场景沉淀成可复用、可治理、可多渠道部署的 Agent 产品能力。

### 页面文案

- 一个平台支撑客户服务、客户经理辅助、销售激活、运营流程和内部生产力场景
- 可复用的 Agent 能力：规划、执行、记忆、工具、技能、子 Agent 和自检
- 可部署到 CRM、IM、Web、桌面端、浏览器和自动化工作流等多个入口
- 从“AI 助手”逐步演进为“AI 工作力”，无需反复重建底层能力

### 视觉建议

使用桥接图：

`单点试点场景 -> 可复用能力层 -> 多场景 Agent 产品`

### 讲解备注

这一页不要展开技术细节，重点让高管理解：我们不是只做一个聊天机器人，而是把每一个成功试点沉淀成后续场景可复用的平台能力。

## 第 2 页

### 标题

不是聊天界面，而是完整的任务执行闭环

### 核心信息

Sage Agent 可以规划、行动、验证并交付结果，因此适合银行里的复杂业务流程，而不只是回答问题。

### 页面文案

- 规划：将复杂需求拆解成可执行步骤
- 执行：调用工具、检索知识、操作文件、触发流程、连接外部系统
- 验证：自检结果、处理异常、让执行进度可见
- 交付：生成材料、更新系统、发送消息、交接给人工团队

### 视觉建议

使用循环图：

`规划 -> 执行 -> 观察 -> 自检 -> 交付`

中心放置：`Human-in-the-loop control`

### 讲解备注

银行里的 Agent 不能只停留在“建议怎么做”。更关键的是，它能不能稳定完成一个过程，例如整理客户信息、生成沟通话术、触达客户、收集反馈、把结果回写到系统。

## 第 3 页

### 标题

工具丰富，但扩展可控

### 核心信息

Sage 支持大量工具和外部能力，同时让每一次 Agent 会话保持聚焦、权限清晰、过程可控。

### 页面文案

- 统一工具层整合内置工具、MCP 服务、浏览器自动化、搜索、图像生成、问卷流程和 IM 触达
- Skills 将业务经验封装为可复用流程，而不是散落在提示词中
- Tool suggestion 会为当前任务选择最相关的工具
- Runtime tool expansion 支持 Agent 在任务中途按需申请更多已授权工具
- 会话级白名单保证工具扩展始终在 Agent 的权限边界内

### 视觉建议

使用两层能力图：

上层：`当前 Agent 任务上下文`

中层：`工具推荐 + 运行时工具扩展`

底层：`已授权能力池：银行 API / CRM / 知识库 / 搜索 / IM / 浏览器 / 文档`

### 讲解备注

这里可以讲一个很小但很关键的差异点：工具很多不是问题，关键是不能把所有工具一股脑塞给模型。Sage 会先为当前任务选择最相关的工具；如果任务中途需要更多能力，Agent 可以申请扩展，但只能在已授权范围内扩展。

## 第 4 页

### 标题

Skills 让银行场景可复用、可维护

### 核心信息

Sage 将平台能力和业务场景知识分离，使新的银行 Agent 产品可以更快组装，也更容易长期维护。

### 页面文案

- Skills 可以封装业务流程、模板、脚本和资产
- 场景逻辑可独立升级，不需要改动核心 Agent 运行时
- 子 Agent 可以按角色分工：客户经理助手、合规审查、活动运营、知识研究
- 可复用 Skills 降低不同分行、渠道和业务线之间的重复建设

### 视觉建议

展示一个 Skills library 支撑多个 Agent：

`KYC Skill / 产品推荐 Skill / 客户跟进 Skill / 合规检查 Skill / 活动复盘 Skill`

### 讲解备注

对银行客户很重要的一点是：业务规则、产品策略和合规要求会不断变化。我们不希望每次都重写 Agent。Skills 的价值是把业务动作沉淀成可维护、可复用的资产。

## 第 5 页

### 标题

面向生产环境的稳定性与管控能力

### 核心信息

Sage 面向真实生产运营设计。对银行来说，可靠性、可视化和治理能力，与模型智能同样重要。

### 页面文案

- 沙箱执行能力让文件、命令和流程操作更安全
- 可视化 Workbench 让 Agent 动作、文件、工具输出和生成物可检查
- Memory 支持会话连续性、用户级连续性和工作区检索
- 可观测性和进度反馈帮助团队复盘执行过程并持续优化
- 共享服务架构支持 Web、桌面端、CLI 和浏览器扩展等多端部署

### 视觉建议

使用三列结构：

`安全` / `可见` / `连续`

每列放 2 个短要点。

### 讲解备注

这一页是给 IT 和风控团队的定心丸。不要只说“我们有大模型”，要说平台如何让 Agent 的执行过程可见、可控、可复盘。

## 第 6 页

### 标题

为什么 Sage 能构建超越单场景竞品的 Agent 产品

### 核心信息

Sage 的优势不是某一个模型或某一个工作流，而是工具、Skills、记忆、编排和产品入口共同形成的复利效应。

### 页面文案

| 维度 | 常见单场景 Agent 方案 | 基于 Sage 的 Agent 平台 |
| --- | --- | --- |
| 场景覆盖 | 面向单一流程定制 | 可复用于客服、客户经理、销售、运营和后台场景 |
| 工具接入 | 固定集成 | 统一工具层 + MCP + 权限内运行时扩展 |
| 业务逻辑 | 依赖大量提示词 | Skills 封装为可维护的业务流程包 |
| 复杂任务 | 单助手循环 | 规划、子 Agent、自检和交付闭环 |
| 运营可见性 | 执行过程难检查 | Workbench、进度反馈、生成物和可观测性 |
| 部署方式 | 单一入口 | Web、桌面端、CLI、浏览器扩展、IM 和嵌入式入口 |

### 讲解备注

这一页可以作为竞品差异总结。重点不要攻击竞品，而是强调“单场景 Agent”和“平台型 Agent 产品”的差异。

## 可选第 7 页

### 标题

从 CMB 经验延展到 某某 的机会

### 核心信息

同样的平台模式可以映射到 某某 的重点业务领域，从高价值、低风险流程开始，逐步扩展到多 Agent 协同运营。

### 页面文案

- 零售银行：客户经理 Next Best Action 助手
- 呼叫中心：问题识别、回复生成和复杂问题升级
- 财富管理：客户画像简报、产品教育和后续跟进计划
- 营销和销售：活动触达、意向识别和线索分配
- 运营流程：文档处理、异常处理和内部流程自动化
- IT 与数字化转型：可治理、可复用的 Agent 工厂

### 讲解备注

这页适合现场互动。建议让客户高管选择他们最想先看的 2-3 个场景，然后我们再进一步展开解决方案。

## 一页式高管摘要

Sage 不是另一个聊天机器人入口，而是一套企业级 Agent 平台。它可以把成功场景转化为可复用能力，并通过完整执行闭环、丰富但可控的工具体系、Skills 场景复用、多 Agent 编排、可视化 Workbench 和多渠道部署，支撑银行从前台客户沟通场景逐步扩展到销售激活、运营流程和内部生产力场景。对 某某 来说，这意味着 Agent 项目可以先从具体高价值场景落地，再逐步形成可治理、可复制的平台能力。

## 会议中的短回答

如果客户问：“你们的 Agent 平台和其他方案有什么不同？”

可以回答：

Sage 构建的 Agent 不只是聊天。它可以规划任务、选择合适工具、在权限范围内扩展工具、委派子 Agent、记住上下文、验证结果，并把最终产物交付到不同渠道。这也是为什么我们可以从 CMB 这样的单个实践场景，扩展为面向 某某 的银行级 Agent 平台。
